Movie Attendance Predictive Analytics

Development Timeline: January 2024 - April 2024

Team: 8 members

In this project, we conducted predictive analysis to determine the likelihood of a person going to the movies based on factors like age, preferred genre, and preferred movie length. The target variable was whether a person would attend a movie, as movie theater profit depends heavily on movie attendance.

We employed multiple machine learning models, including Decision Trees, Artificial Neural Networks (ANNs), and Logistic Regression, to predict movie attendance. Each model was evaluated using performance metrics such as accuracy, precision, recall, and F1-score.

The Decision Tree model identified key factors like age and movie intensity to predict movie attendance, achieving high accuracy in certain nodes. The ANN model, constructed using TensorFlow’s Keras API, provided insights into the relationships between demographic factors and movie attendance over multiple epochs. Logistic Regression offered interpretable results in terms of likelihood and performed well for this binary classification task.

We also utilized data visualization techniques such as confusion matrices, ROC curves, precision-recall curves, correlation matrices, heatmaps, pair plots, violin plots, and data density plots to explore the relationships between variables and assess the performance of each model.

Encoder and Decoder

This project involves the development of a Java-based message encoder and decoder using Substitution Cipher and Shuffle Cipher techniques. The goal was to provide users with the ability to encode and decode messages by selecting a cipher type, inputting a message from a file, and specifying whether the message should be encoded or decoded.

The Substitution Cipher works by shifting each letter in the message by a specified amount, while the Shuffle Cipher scrambles or unscrambles the message a specified number of times. The user can choose between these two methods to process their messages. The system also allows the user to specify input and output file names for reading and saving the encoded or decoded messages.

The project was developed in Java using IntelliJ IDEA, and it incorporates file I/O to handle reading and writing messages to and from text files. Key features include a robust error handling system, the ability to process both encoding and decoding requests, and an easy-to-use interface for selecting cipher options. The project's algorithms are optimized for quick and efficient encoding and decoding, ensuring that even large files can be processed seamlessly.

The main program architecture includes:

  • SubstitutionCipher: Handles encoding and decoding by shifting characters in the message.
  • ShuffleCipher: Provides the functionality to shuffle or unshuffle the message multiple times.
  • File I/O: Facilitates reading input messages from files and writing the encoded/decoded messages to output files.

Hospital Billing System

The Hospital Billing System project was designed to simplify hospital billing processes by allowing input of patient and doctor information, calculating charges, and generating detailed bills. The system captures key data such as patient ID, name, date of birth, admission and discharge dates, attending physician details, and charges associated with services like room, pharmacy, and doctor fees.

The project is built using object-oriented principles in Java, with the following classes playing a central role:

  • Person: A base class that represents individuals with attributes like first name and last name.
  • Doctor: A subclass of Person, which adds the doctor’s specialty as an attribute.
  • Patient: Another subclass of Person, this class captures additional patient-specific information like ID, date of birth, admission/discharge dates, and the assigned physician.
  • Date: Handles date-related functionalities, such as setting and printing admission and discharge dates.
  • Bill: Manages billing details, such as charges for pharmacy, room, and doctor services, and calculates the total bill for each patient.

The system enables the entry of multiple patients and doctors, calculates the total charges for each patient, and generates a bill. Both the patient information and the generated bill are printed to the console and saved to a text file with the patient’s name for easy reference.

The application is user-friendly and provides a loop for continuous input of patient and doctor details until the user decides to stop. The bill generation process is automated, ensuring accuracy in calculations and formatting. By providing both console output and file storage, the system ensures that all data is available for future reference.
